The Dual-Front Challenge for Sustainability

Land Scarcity

The world needs to increase food production by an estimated 60% by 2050, yet available farmland is decreasing due to urbanization and climate change.

Energy Demands

The global demand for clean, renewable energy is skyrocketing. Utility-scale solar farms require vast amounts of land, often competing with agriculture.

How can we produce more food and more energy with less land?

2 / 11

Our Solution

Introducing AVISp

AVISp is a professional-grade simulation platform designed to solve this land-use conflict. It empowers farmers, investors, and policymakers to design and analyze **agro-voltaic systems** — the co-location of solar panels and agricultural crops on the same piece of land.

Simulate. Analyze. Optimize.

Make data-driven decisions before breaking ground.

Core Features

Live Weather API

Integrates with a live weather service to pull real, historical solar irradiance data for any location on Earth.

Dynamic Soil Modeling

Utilizes a validated input system for soil composition, directly impacting water retention and savings calculations.

Real-time Simulation

The entire system recalculates instantly as you change parameters, providing immediate feedback on your design choices.

Interactive 3D View

A dynamic, high-fidelity 3D rendering of the farm layout allows for intuitive visualization and inspection of the proposed system.

Our Design Process

1

Ideation: Identified the critical need for dual-use land solutions and conceptualized a data-driven planning tool.

2

Prototyping: Built the core simulation engine in Python and a basic frontend to validate the concept and core calculations.

3

Iteration & Refinement: Faced challenges with user input, such as our initial soil triangle concept. We pivoted to a more robust, validated text input system to improve usability and reliability.

4

Integration: Integrated external systems like the live weather API and the 3D rendering library to add layers of complexity and real-world value.

5

Testing: Rigorously tested the application for bugs, ensuring a stable and reliable user experience for our final product, AVISp v2.1.

Code Highlight: The Simulation Core

This snippet from app.py shows how we combine user inputs and live data to calculate a result.


# Fetches live weather data for the location
base_solar_irradiance = get_solar_data(latitude, longitude)

# Calculates a water retention factor from user soil input
sand = soil_composition.get("sand", 33) / 100
silt = soil_composition.get("silt", 33) / 100
clay = soil_composition.get("clay", 34) / 100
soil_water_retention_factor = 1.0 + (0.3 * clay) + (0.1 * silt) - (0.15 * sand)

# Retrieves modifiers for the chosen panel and crop
panel_mod = panel_modifiers.get(panel_config)
crop_mod = crop_modifiers.get(crop_type)

# Calculates the final water savings percentage
water_saved = 100 - (100 * (1 - panel_mod['shading_factor']) / soil_water_retention_factor)

This modular approach allows for easy updates and expansion of our scientific models.
